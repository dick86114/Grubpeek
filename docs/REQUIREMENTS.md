# GrubPeek 需求文档

## 1. 项目简介
GrubPeek（今天有什么好吃的）是一个企业级食堂每日菜单展示系统。该系统旨在为员工提供便捷的食堂菜谱查询服务，支持多餐别（早餐、午餐、晚餐、外卖/包点）展示，并具备后台管理功能，允许管理员上传和维护菜单数据。

## 2. 目标用户
- **普通员工**：查看每日菜谱、特色菜品、外卖信息。
- **系统管理员**：上传菜单文件（Excel/WPS）、管理每日菜单详情、系统设置。

## 3. 功能需求

### 3.1 用户端功能
1.  **首页展示**：
    *   展示当天的日期、星期及天气/节日氛围（通过UI体现）。
    *   **每日菜单卡片**：按餐别（早餐、午餐、晚餐、外卖包点）分卡片展示。
    *   **菜品详情**：展示菜品名称、是否特色菜（星标高亮）、价格（部分餐别）。
    *   **特色菜推荐**：突出显示“档口特色”或“推荐菜品”。
2.  **日历导航**：
    *   提供月视图日历。
    *   用户可点击日历切换查看历史或未来日期的菜单。
    *   日历上应标记有菜单数据的日期。
3.  **主题切换**：
    *   支持浅色（Light）、深色（Dark）及跟随系统（System）三种模式。
    *   界面风格需温馨、现代，适配不同光照环境。

### 3.2 管理端功能
1.  **身份验证**：
    *   简单的密码登录机制，保护管理后台入口。
2.  **菜单管理**：
    *   **文件上传**：支持上传 `.xlsx` 或 `.et` 格式的菜单文件。
    *   **批量导入**：系统自动解析上传的文件，提取日期、餐别、菜品信息并存入数据库。
    *   **数据列表**：按周或文件维度展示已导入的菜单记录。
    *   **删除功能**：支持删除特定时间段的菜单数据。
3.  **每日菜单编辑**（高级）：
    *   支持对单日菜单进行增删改查（虽然主要依赖Excel导入，但保留手动修正能力）。
4.  **系统设置**：
    *   修改管理员密码。

## 4. 非功能需求
1.  **性能**：页面加载速度快，日历切换无延迟。
2.  **兼容性**：支持主流浏览器（Chrome, Edge, Safari, Firefox）及移动端访问（响应式布局）。
3.  **易用性**：界面直观，操作简单，无需复杂培训即可上手。
4.  **数据安全**：管理后台需密码访问，防止未授权修改。

## 5. 数据需求
-   **菜单数据结构**：包含日期、餐别（早/午/晚/外卖）、分类（如主食、热菜、凉菜、档口特色）、菜名、价格、是否特色。
-   **数据源**：主要来源于行政部门提供的 Excel 表格，格式相对固定但需具备一定的容错解析能力。
